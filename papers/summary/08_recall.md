# `recall` — ReCaLL (Relative Conditional Log-Likelihood)

**Primary source:** Roy Xie, Junlin Wang, Ruomin Huang, Minxing Zhang, Rong Ge, Jian Pei, Neil
Zhenqiang Gong, Bhuwan Dhingra (Duke University). *ReCaLL: Membership Inference via Relative
Conditional Log-Likelihoods.* EMNLP 2024 (main), pp. 8671–8689. arXiv:2406.15968.

## Brief overview

ReCaLL detects LLM training data via the **relative change in a sequence's conditional log-likelihood
when it is prepended with a non-member prefix/context**. The empirical finding is that conditioning
*member* data on a non-member prefix induces a larger decrease in log-likelihood than for *non-member*
data. The paper interprets this through "in-context learning as implicit fine-tuning": the prefix
perturbs the model's already-memorized predictive distribution for members more than for unfamiliar
non-members. It is reference-model-free, inference-time only, and members receive higher ReCaLL
scores.

## Detailed attack mechanism

Given an autoregressive model `M` and a target `x`, compute a membership score and threshold it.

- **Prefix construction:** `P = p₁ ⊕ p₂ ⊕ … ⊕ p_n`, a concatenation of `n` data points ("shots")
  *known to be non-members* — obtainable via a knowledge-cutoff date, user-generated text, or
  synthetic generation. For a given dataset the prefix `P` is **fixed** across all targets.
- For each target `x` compute two log-likelihoods from `M`: the unconditional `LL(x)` and the
  conditional `LL(x | P)`.
- **ReCaLL score:** `ReCaLL(x) = LL(x | P) / LL(x)`.
- Since log-likelihoods are negative and conditioning lowers the LL more for members
  (`LL(x_m | P) < LL(x_m)`), members get scores typically greater than 1 and higher than non-members:
  `E[ReCaLL(x_member)] > E[ReCaLL(x_non-member)]`. Threshold the score; **higher ⇒ member**.

Practical findings: the only hyperparameter is `n` (number of shots); even a **single shot** beats
all baselines, and performance rises with more shots until the context window is exhausted. A
**fixed** prefix beats per-target dynamic prefixes; most-TF-IDF-similar / same-domain prefixes work
best, random next, and *member* prefixes work poorly (and would require training-data access). The
largest log-likelihood changes occur in the beginning tokens. An **ensemble** variant splits many
shots into smaller prefix sets, computes a ReCaLL score per set, and averages — reducing variance and
sidestepping context-window limits.

## Attack implementation

- **Code repository (verified):** https://github.com/ruoyuxie/recall (project page:
  royxie.com/recall-project-page). Run via `src/run.py` with `--target_model`, `--ref_model`,
  `--dataset wikimia`, `--sub_dataset {32,64,128}`, `--num_shots N`, and flags `--pass_window`,
  `--synthetic_prefix`, `--api_key_path`.
- **Benchmarks:** WikiMIA (length splits 32/64/128) and MIMIR (Pile, with 7-gram & 13-gram filtering
  for minimal member/non-member distribution shift).
- **Models:** WikiMIA — Pythia-6.9B, GPT-NeoX-20B, LLaMA-30B, OPT-66B, Mamba; MIMIR — Pythia family
  (160M–12B). Synthetic prefixes generated by GPT-4o. **Baselines:** LOSS, Reference, Zlib, Neighbor,
  Min-K%, Min-K%++. **Metrics:** AUC (main) and TPR@1%FPR. Result: SOTA on WikiMIA, beating runner-up
  Min-K%++ by ~14.8–15.4% AUC across lengths, and competitive on the harder MIMIR; it works even with
  random or synthetic prefixes.

## Adaptation for LLM fine-tuning + data extraction

ReCaLL was developed for the pre-training / single-epoch setting (the harder regime). For a
fine-tuned LLM the mechanism transfers naturally: build a fixed non-member prefix `P` from text the
attacker knows is outside the fine-tuning set (post-cutoff or synthetic), then score candidate
fine-tuning samples by `ReCaLL(x) = LL(x | P) / LL(x)`; a higher relative conditional log-likelihood
flags fine-tuning members. The paper notes the multi-epoch fine-tuning regime is generally an
*easier* MIA setting because of stronger memorization, so the signal should be stronger. The
in-context-learning-as-fine-tuning interpretation underpins this: the prefix perturbs memorized
(member) examples more.

It is practical for fine-tuned models because non-member prefixes are easy to obtain (post-cutoff or
synthetic), and GPT-4o-generated synthetic prefixes give near-equal performance, so the attack works
even without ground-truth non-members. For data extraction, ReCaLL is positioned as the membership-
confirmation primitive underlying extraction and memorization auditing (detecting copyrighted text,
PII, or benchmark/test data in the training set): confirming that a candidate string is a member
surfaces/verifies memorized data, which a generation step can then attempt to reconstruct.
