# `spv_mia` — SPV-MIA (Self-calibrated Probabilistic Variation)

**Primary source:** Wenjie Fu, Huandong Wang, Chen Gao, Guanghua Liu, Yong Li, Tao Jiang.
*Practical Membership Inference Attacks against Fine-tuned Large Language Models via Self-prompt
Calibration.* NeurIPS 2024. arXiv:2311.06062. Code: https://github.com/tsinghua-fib-lab/NeurIPS2024_SPV-MIA
(legacy: github.com/wjfu99/MIA-LLMs).

## Brief overview

SPV-MIA is purpose-built for the **fine-tuned-LLM, black-box, practical** setting. It argues that
prior MIAs underperform on real LLMs for two reasons: reference-based attacks need a reference dataset
that closely resembles the private training set (usually unavailable), and almost all attacks assume
the target model is *overfitted*, whereas released LLMs are regularised and merely *memorize*. SPV-MIA
fixes both with two modules: (1) a **self-prompt reference model** trained on text generated by the
target LLM itself (removing the need for a real reference dataset), and (2) a **probabilistic-variation**
membership signal based on memorization rather than overfitting. It raises MIA AUC from ~0.7 to ~0.9
across four LLMs and three datasets (about a 23.6% average AUC improvement over baselines).

## Detailed attack mechanism

**Threat model:** black-box access to a target causal LM `θ` fine-tuned on a private `D_mem`; the
adversary may fine-tune a reference model on a disjoint same-task dataset. Existing attacks are framed
as: reference-free `A(x,θ) = 1[p_θ(x) ≥ τ]`; reference-based
`A(x,θ) = 1[p_θ(x) − p_φ(x) ≥ τ]` (difficulty calibration, Watson et al.).

**(1) Practical Difficulty Calibration via a self-prompt reference model.** Difficulty calibration
needs `Δm(x) = m_θ(x) − E_{φ}[m_φ(x)]` with a reference model `φ` trained on a dataset drawn from the
target's distribution — which is unrealistic for private data. SPV-MIA instead collects short text
chunks (length `l`) from a public same-domain dataset, uses each as a prompt to make the **target LLM
generate** text, and fine-tunes a reference model `θ̇` on the resulting self-generated dataset
`D_self ∼ p_θ(x)`. The calibrated metric becomes `Δm(x) ≈ m_θ(x) − m_θ̇(x)`. This approaches the
performance of an identical-distribution reference while needing only a few prompt tokens, and is
robust even with as few as 1,000 queries or completely unrelated prompts.

**(2) Probabilistic Variation Assessment via symmetrical paraphrasing.** Rather than thresholding raw
probability, SPV-MIA detects whether a record sits at a **local maximum** of the model's probability
surface (the signature of memorization, present before overfitting). Using a second-partial-derivative
test, it defines the probabilistic variation as the expected second-order directional derivative,
approximated symmetrically:

`p̃_θ(x) ≈ (1/2N) Σ_n ( p_θ(x̃_n⁺) + p_θ(x̃_n⁻) ) − p_θ(x)`,

where `x̃_n^± = x ± z_n` is a pair of symmetric paraphrases of `x` produced by a paraphrasing model
(either by adding/subtracting Gaussian noise in the embedding domain, or by masking ~20% of tokens
and reconstructing with T5 in the semantic domain). The final decision combines both modules:
`A(x, θ, θ̇) = 1[ p̃_θ(x) − p̃_θ̇(x) ≥ τ ]`. This reformulation also re-derives the neighbourhood
attack (Mattern et al. 2023) under a more rigorous principle, without requiring a separate LLM as in
the original.

## Attack implementation

- **Code repository:** https://github.com/tsinghua-fib-lab/NeurIPS2024_SPV-MIA (official; legacy at
  github.com/wjfu99/MIA-LLMs). Pipeline: fine-tune the target model
  (`ft_llms/llms_finetune.py`), generate the self-prompt reference dataset (`refer_data_generate.py`),
  fine-tune the self-prompt reference model (fewer epochs), then `attack.py`.
- **Models:** GPT-2, GPT-J (6B), Falcon-7B, LLaMA-7B. **Datasets:** Wikitext-103, AG News, XSum.
  **Fine-tuning:** LoRA by default (also Prefix-Tuning, P-Tuning, (IA)³), 10 epochs for the target /
  4 for the reference, LR 1e-4, AdamW with early stopping (to avoid overfitting). **Baselines:** Loss,
  Neighbour, DetectGPT (reference-free) and LiRA-Base, LiRA-Candidate (reference-based).
- **Result:** average AUC 0.924 across all settings (e.g. 0.951 on Wikitext for LLaMA), ~30% over the
  strongest baseline (LiRA-Candidate), which itself assumes full access to an auxiliary dataset.

## Adaptation for LLM fine-tuning + data extraction

This attack *is* the fine-tuning adaptation — it targets the fine-tuning phase directly, which the
paper identifies as the most privacy-sensitive stage because fine-tuning datasets are small and
private. Its design choices make it the most "practical" of the ten: black-box only (needs log-probs
/ generations, not weights), no real reference dataset (the reference is bootstrapped from the
target's own generations), and a memorization-based signal that works even when the model is not
overfitted. The paper shows it remains effective across PEFT methods (AUC rising with the number of
trainable parameters) and even under DP-SGD at moderate privacy budgets (e.g. AUC ~0.87 at `ε = 60`,
falling to ~0.77 at `ε = 15`), which is a direct, quantitative privacy-assessment result: stronger DP
reduces but does not eliminate the leakage.

For data extraction, SPV-MIA's self-prompt step already generates text from the target model, and its
membership signal identifies which records the model has memorized. The high-`p̃_θ` records are local
maxima of the model's distribution — the records the model will most readily regenerate — so the
attack both confirms membership and points to the fine-tuning content most amenable to verbatim
extraction. Its main stated limitation is that it is designed for causal LMs; extension to MLM/Seq2Seq
models is left as future work.

## Relation to the project's main paper

The main project paper studies an *active* attacker against *federated vision* models under local
differential privacy. SPV-MIA is the closest of the ten to that privacy-assessment goal in the LLM
fine-tuning environment: it provides a passive, black-box leakage measurement and an explicit
DP-SGD-vs-AUC trade-off, making it a natural instrument for assessing whether privacy has improved or
declined in a fine-tuned-LLM environment (e.g., before vs. after adding DP, changing PEFT method, or
applying a defense).
